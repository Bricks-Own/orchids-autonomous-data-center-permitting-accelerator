Reference: https://ui.shadcn.com/create?preset=b1zM2TW91E&template=vite-monorepo&pointer=true&rtl=true — the same preset already used for this whole redesign. index.css already defines --chart-1 through --chart-5 (mapped to --color-chart-1 etc.) in the correct lime/emerald hue family — every chart in the app should use only these, not arbitrary reds/blues/grays.

permitOS/src/components/MilestoneTimeline.jsx:

Sweep the codebase for any stroke="#, fill="#, color="#, or bg-red-/bg-blue-/bg-amber-/etc. used specifically in chart/data-viz contexts, and replace with the theme's --chart-1 through --chart-5 variables (or --primary/--muted-foreground where the color is meant to convey emphasis rather than category). Grid lines (CartesianGrid stroke) should use var(--border) instead of a hardcoded #1f2937.

PART 2 — Remove unnecessary horizontal scrolling

The Gantt chart in MilestoneTimeline.jsx wraps its content in overflow-x-auto with a hardcoded minWidth: '900px' (around line 290) — this forces a horizontal scrollbar on any viewport where the content area (sidebar + main content) is narrower than 900px, which is common at normal desktop widths. Since all bar positions are already computed as percentages of ganttWeeks (not fixed pixels), the fixed minWidth isn't actually needed for the chart to render correctly — remove it and let the chart scale to its actual container width. If any label truncation becomes a problem at narrow widths once the fixed width is removed, shrink the row-label column width slightly rather than reintroducing a scroll container.

Verify: check every page this session touched (Site Intake, Permits, Milestone Timeline) at a normal desktop width (around 1280-1440px) and confirm there's no horizontal scrollbar anywhere that isn't genuinely necessary. If there's some other page with an unwanted horizontal scrollbar found during this sweep, fix that too and note what it was.

PART 3 — Add a dynamic phase timeline (sequential, proportional-width, not just a bar chart)

The two PhaseBreakdownCharts (Traditional / Brick-Accelerated) already show phase durations as a horizontal bar chart, but that's a category-axis chart, not an actual timeline — add a genuine sequential timeline view alongside it: a horizontal line of connected phase markers where the spacing between markers is proportional to how many weeks that phase actually consumes, so you can see at a glance which phases eat the most time, not just read it off a bar length.

Base this on the same visual language already used in permitOS/src/components/Stepper.jsx (dot + label + connecting line between dots), but make segment widths proportional instead of equal, and add each phase's duration as a label under its segment. Create permitOS/src/components/PhaseTimeline.jsx:

import React from 'react';

export default function PhaseTimeline({ phases, totalWeeks, accentColor = 'var(--primary)' }) {
  return (
    <div className="w-full overflow-hidden">
      <div className="flex items-stretch w-full">
        {phases.map((phase, i) => {
          const widthPct = (phase.weeks / totalWeeks) * 100;
          return (
            <div
              key={phase.label}
              className="flex flex-col items-center min-w-0"
              style={{ width: `${widthPct}%` }}
            >
              <div className="flex items-center w-full">
                <div
                  className="w-2.5 h-2.5 rounded-full flex-shrink-0"
                  style={{ background: i === 0 ? accentColor : 'var(--muted-foreground)' }}
                />
                {i < phases.length - 1 && (
                  <div className="h-px flex-1" style={{ background: 'var(--border)' }} />
                )}
              </div>
              <div className="mt-2 text-center px-1">
                <div className="text-xs font-medium text-foreground truncate w-full" title={phase.label}>
                  {phase.label}
                </div>
                <div className="text-xs text-muted-foreground mt-0.5">{phase.weeks}wk</div>
              </div>
            </div>
          );
        })}
      </div>
    </div>
  );
}

Render two of these in MilestoneTimeline.jsx, right above (or below — whichever reads better) the two PhaseBreakdownCharts, one for Traditional (phases={traditionalPhases} totalWeeks={comparison.traditional.totalWeeks}) and one for Brick-Accelerated (phases={brickPhases} totalWeeks={comparison.brickAccel.totalWeeks}), each labeled with which pathway it represents. This gives users both views: the bar chart for comparing exact durations, and this timeline for seeing the sequence and relative time-weight of each phase at a glance.
