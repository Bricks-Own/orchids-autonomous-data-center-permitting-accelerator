Redesign Site Planner to match the rest of the app's shadcn Sera theme
permitOS/src/components/SitePlanner.jsx is the only page in the app that doesn't use real shadcn components. It imports zero ./ui/* components — every card is a raw <div className="border border-border/40 bg-card/40 p-4">, every button is a raw <button>, and the sliders are raw <input type="range"> instead of the shadcn Slider component already used elsewhere (e.g. SiteIntake.jsx's Brick Load Reduction slider). It also has two panels using a violet-950 accent color that doesn't exist anywhere else in the app — every other page uses only the emerald/lime theme colors.

Use this exact shadcn preset, same as the rest of the app: https://ui.shadcn.com/create?preset=b1zM2TW91E&template=vite-monorepo&pointer=true&rtl=true&item=preview
(Sera style, Stone base, Emerald theme, Lime chart color, Raleway headings — sharp corners, no rounded-*.)

Required changes
1. Use real shadcn components everywhere
Replace every hand-rolled <div className="border ... bg-card/40 p-4"> with Card / CardHeader / CardTitle / CardContent (same pattern as SiteIntake.jsx and DocumentFactory.jsx). Replace every raw <button> with Button (variant="default" | "secondary" | "outline" | "ghost", matching how they're used elsewhere). Replace the raw <input type="range"> sliders in ScenarioTest with the shadcn Slider component. Replace the SMALL/MID/LARGE/CA text badges on the Scenario Explorer cards with the Badge component.

2. Kill the violet accent — theme colors only
The "Quick Scenario Test" and "Custom Scenario Builder" panels use bg-violet-950/20 and text-violet-300. Remove this entirely — use the same card/primary styling as every other panel on the page. Also fix the map's site-boundary rectangle, which is hardcoded to #6366f1 (indigo) in SiteMap's updateBoundary — it should use the theme's chart color (var(--color-chart-1) or similar) like every other visual element in the app.

3. Reduce visual complexity — use Tabs to organize, don't stack everything
Right now the page stacks: location presets → map → site config summary → two buttons → a togglable scenario panel → a second column with 4 scenario cards → a second togglable scenario panel (duplicate of the first) → a reference legend. That's too much simultaneously visible with no hierarchy.

Restructure into two clear zones:

Left: Map + location presets + current site config, as the primary "where and what" view.
Right: A Tabs component with two tabs — "Scenario Explorer" (the 4 preset cards) and "Custom Builder" (the slider-based ScenarioTest) — instead of two separate always-present togglable panels that duplicate each other. This removes the redundant "Quick Scenario Test" / "Custom Scenario Builder" panels entirely — there should be exactly one place to adjust custom parameters, not two.
4. Fix the redundant click target on Scenario Explorer cards
Each scenario card is currently both (a) clickable as a whole card, which only previews the scenario on the map without navigating, and (b) contains a nested "Apply → Intake" button that applies and navigates. Two overlapping interactions on one card is confusing. Pick one: make the whole card clickable to preview (as now), and keep a single, clearly-separated "Apply → Intake" button below/outside the card body — not nested inside a cursor-pointer click target — so there's no ambiguity about what a click on the card vs. the button does.

5. Match the page header style used everywhere else
Every other page (Site Intake, Document Factory, etc.) uses a plain heading: <h1 className="text-4xl font-bold text-foreground tracking-[-0.02em]"> with a <p className="text-sm text-muted-foreground"> subtitle — no colored box. Site Planner currently wraps its title in a border-primary/30 bg-primary/10 colored banner box, which is inconsistent with the rest of the app. Change it to match the plain heading style, and move the "Open Full Site Intake" button to a normal Button in the header row (same place Document Factory/Milestone Timeline put their header actions).

6. Clean up the "Current Site Configuration" grid
Replace the current 8-cell bg-muted/40 p-2.5 grid with the same stat-card pattern already used in Document Factory / Milestone Timeline (small Card components with a label and value), so it's visually consistent with stat displays elsewhere in the app instead of looking like a separate, ad-hoc component.

What NOT to change
Keep the Leaflet map, its street/satellite toggle, draggable marker, and click-to-reposition behavior exactly as-is — only restyle the surrounding chrome (legend row, coordinate inputs) with real shadcn components.
Keep all the existing scenario data (Edge/Colocation/Hyperscale/CA Nonattainment presets and their params) exactly as-is — this is a visual/structural cleanup, not a data or logic change.
Don't touch the Site Planner → Site Intake handoff logic (navigateToIntake, the fromPlanner banner) — that was just fixed and verified working; leave it alone.
Goal
After this change, Site Planner should look and feel like it belongs to the same product as Site Intake and Document Factory — real Cards, real Buttons, one consistent accent color, and a Tabs-based layout that shows one clear thing at a time instead of nine stacked panels.
