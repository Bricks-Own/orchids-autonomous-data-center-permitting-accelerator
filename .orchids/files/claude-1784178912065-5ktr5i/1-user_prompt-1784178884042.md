Site Intake rework: fix padding, remove upfront permit picking, consolidate permit pages, real charts

Part 1 — Fix missing padding on the Review step (Step 3) of Site Intake
In permitOS/src/components/SiteIntake.jsx, every card in renderStep3() renders its content directly as a child of <Card> instead of wrapping it in <CardContent>. Look at ui/card.jsx: Card only has vertical padding (py-(--card-spacing)), horizontal padding (px-(--card-spacing)) only comes from CardContent. That's why Step 3's inner content sticks flush to the card edges, unlike Step 1 and Step 2 which correctly use <Card><CardContent>...</CardContent></Card>.

Fix: wrap the contents of all 7 <Card> blocks in renderStep3() (Site Summary, Permitting Timeline Comparison, and each of the Air/Water/Building/Power permit breakdown cards, plus the Permitting Phases Overview card) in <CardContent>, exactly matching the pattern already used correctly in renderStep1() and renderStep2(). Don't change any of the inner content or logic — purely wrap each card's existing children in <CardContent>...</CardContent>.

Part 2 — Remove the upfront "pick your permit types" step; derive it instead
Right now Step 1 makes users manually check boxes for "Air Permit," "Water Permit," etc. before they've entered any site details. That's backwards — a team that's just starting to plan a data center often doesn't know yet which permits they'll need; that's supposed to be something this tool figures out for them, not something they self-diagnose upfront using EPA jargon.

Replace the "Permit Types Needed" card in renderStep1() with a plain-English Q&A that doesn't require the user to already know permitting terminology:

const PROJECT_QUESTIONS = [
  { key: 'hasOnSiteGeneration', question: 'Will this site include on-site power generation — turbines, generators, or other combustion equipment?', helps: 'air' },
  { key: 'hasWaterUse', question: 'Will this site use or discharge water — cooling towers, process water, wastewater?', helps: 'water' },
  { key: 'hasNewConstruction', question: 'Is this new building construction, or a build-out inside an existing structure?', helps: 'building' },
  { key: 'hasGridInterconnection', question: 'Will this site need a new or upgraded grid interconnection?', helps: 'power' },
];
Render these as Yes/No toggle cards (reuse the same button/card visual style already used for the Project Type selector right below it — border, icon, label, checkmark-when-selected). Store answers on inputs (e.g. inputs.hasOnSiteGeneration, etc., default true for all so nothing breaks for users who skip ahead).

Also add a clearly visible secondary option on this same step: "Not sure yet? Plan your data center first" — a text link or outline button that calls setActiveTab('siteplanner'), for users who don't have their equipment specs figured out yet (SitePlanner.jsx already accepts setActiveTab and routes back to 'intake' when the user applies a scenario, so this is just adding the reverse link).

Then derive permitTypesNeeded automatically instead of reading it from a manual checkbox array. Replace the permitTypes derivation (currently inputs.permitTypesNeeded || ['air','water','building','power']) with a computed value based on the Q&A answers AND the actual entered equipment values, so it stays accurate even if someone changes their answer later:

const derivedPermitTypes = [
  (inputs.hasOnSiteGeneration !== false || inputs.turbines > 0 || inputs.gensetCount > 0) && 'air',
  (inputs.hasWaterUse !== false || inputs.coolingMGD > 0 || inputs.waterMGD > 0) && 'water',
  inputs.hasNewConstruction !== false && 'building',
  inputs.hasGridInterconnection !== false && 'power',
].filter(Boolean);
Use derivedPermitTypes everywhere permitTypes is currently used (Step 2's conditional section rendering, Step 3's permit breakdown cards, the sidebar's permitKey filtering in App.jsx). Remove the old togglePermitType function and PERMIT_TYPE_CONFIG-driven checkbox UI entirely — that whole concept goes away.

Part 3 — Consolidate the 4 separate permit pages into 1 "Permits" page
Right now App.jsx has 4 separate sidebar items — Air Permit AI, Water Permit AI, Building Permitting, Power Permitting — each a full top-level tab (AirPermitAI.jsx, WaterPermitAI.jsx, BuildingPermitAI.jsx, PowerPermitAI.jsx). Consolidate these into one sidebar item, "Permits."

Create permitOS/src/components/Permits.jsx:

import React, { useState } from 'react';
import AirPermitAI from './AirPermitAI';
import WaterPermitAI from './WaterPermitAI';
import BuildingPermitAI from './BuildingPermitAI';
import PowerPermitAI from './PowerPermitAI';
// reuse existing Card/CardContent/Button and Wind/Drop/Buildings/Lightning icons

const PERMIT_DEFS = {
  air: { label: 'Air Permit', icon: Wind, Component: AirPermitAI },
  water: { label: 'Water Permit', icon: Drop, Component: WaterPermitAI },
  building: { label: 'Building Permit', icon: Buildings, Component: BuildingPermitAI },
  power: { label: 'Power / Interconnection', icon: Lightning, Component: PowerPermitAI },
};

export default function Permits({ inputs, results, setActiveTab }) {
  const [selected, setSelected] = useState(null);

  if (!results) {
    return (
      <div className="px-10 py-8 max-w-[1180px] mx-auto">
        <Card><CardContent className="text-center py-16">
          <p className="text-lg font-semibold text-foreground mb-2">No permits generated yet</p>
          <p className="text-sm text-muted-foreground mb-6">Run Site Intake to generate the permits this site needs, or plan your project first if you're not sure what you need.</p>
          <div className="flex justify-center gap-3">
            <Button onClick={() => setActiveTab('intake')}>Generate via Site Intake</Button>
            <Button variant="outline" onClick={() => setActiveTab('siteplanner')}>Plan via Site Planner</Button>
          </div>
        </CardContent></Card>
      </div>
    );
  }

  const permitTypes = inputs?.permitTypesNeeded || ['air', 'water', 'building', 'power'];

  if (selected) {
    const { Component, label } = PERMIT_DEFS[selected];
    return (
      <div className="px-10 py-8 max-w-[1180px] mx-auto space-y-4">
        <Button variant="outline" onClick={() => setSelected(null)}>&larr; Back to Permits</Button>
        <Component results={results} inputs={inputs} setActiveTab={setActiveTab} />
      </div>
    );
  }

  return (
    <div className="px-10 py-8 max-w-[1180px] mx-auto">
      <h1 className="text-4xl font-bold text-foreground mb-6">Permits</h1>
      <div className="grid grid-cols-2 gap-5">
        {permitTypes.map(key => {
          const { label, icon: Icon } = PERMIT_DEFS[key];
          return (
            <Card key={key} className="cursor-pointer hover:bg-card/80" onClick={() => setSelected(key)}>
              <CardContent className="flex items-center gap-3">
                <Icon weight="duotone" size={24} className="text-muted-foreground" />
                <span className="text-base font-semibold text-foreground">{label}</span>
              </CardContent>
            </Card>
          );
        })}
      </div>
    </div>
  );
}
