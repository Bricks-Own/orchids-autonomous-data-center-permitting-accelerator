 fix broken inputs in Site Intake + increase Brick timeline benefit

Part 1 — Every typed input field in Site Intake is broken.

In permitOS/src/components/SiteIntake.jsx, all 29 <Input> usages call onChange={v => update('field', v)}, treating v as the raw typed value. But Input here is imported from ./ui/input.jsx (the shared shadcn/Base UI component), which passes through the native onChange event unmodified — it does not extract .target.value for you. So v is actually the full React SyntheticEvent object, and every keystroke sets that field to an event object instead of a string/number. This is why typing a new number into any field (turbine count, MW per turbine, heat rate, etc.) does nothing.

Confirmed this is scoped to this file only — AuthModal.jsx already uses the same shared Input component correctly with onChange={e => setName(e.target.value)}, so fix the call sites in SiteIntake.jsx, do not change ui/input.jsx itself (that would break AuthModal.jsx).

Fix: for every <Input ... onChange={v => ...}> in SiteIntake.jsx, change the parameter from v to e, and replace every reference to the raw value with e.target.value. For fields with type="number", wrap it in parseFloat(e.target.value) || 0 (matching the number-parsing behavior the old pre-redesign local Input component used to do internally).

Examples of the exact transformation:

// text field — before
<Input value={inputs.siteName} onChange={v => update('siteName', v)} />
// text field — after
<Input value={inputs.siteName} onChange={e => update('siteName', e.target.value)} />

// number field — before
<Input value={inputs.turbines} onChange={v => update('turbines', v)} type="number" />
// number field — after
<Input value={inputs.turbines} onChange={e => update('turbines', parseFloat(e.target.value) || 0)} type="number" />

// field with a side effect — before
<Input value={inputs.county} onChange={v => { update('county', v); markAddressTouched(); }} />
// field with a side effect — after
<Input value={inputs.county} onChange={e => { update('county', e.target.value); markAddressTouched(); }} />
Apply this same transformation to all 29 <Input> onChange handlers in the file — every field in Site Information, Generation Equipment, Data Center Systems, Water Systems, Building Permitting, and Power Permitting sections. Number-type fields: siteAcres, turbines, mwPerTurbine, hours, datacenterMW, pueTarget, phases, heatRate, noxFactor, coFactor, stackHeight, nearestReceptorFt, coolingMGD, blowdownPct, waterMGD, buildingSqFt, stories, interconnectionVoltage, transformerCapacity, gensetCount, gensetHP, gensetHours. Text fields (no parseFloat): siteName, client, county, address, codTarget, lat, lon.

Part 2 — Increase the Brick-Accelerated timeline benefit.

In permitOS/src/utils/timelineCalc.js, computePathwayDuration(), increase the emissions-reduction coefficient from 0.25 to 1.0:

// before
const reductionMul = 1 - Math.min(Math.max(emissionsReductionPct, 0), 0.4) * 0.25;
// after
const reductionMul = 1 - Math.min(Math.max(emissionsReductionPct, 0), 0.4) * 1.0;
Leave everything else in that file untouched. Verify: run screening on the default BigWatt/Tennessee demo site (20% Brick reduction) and confirm Traditional vs. Brick-Accelerated now shows roughly 13 vs 10 months (~23% saved), not the previous 13 vs 11.
