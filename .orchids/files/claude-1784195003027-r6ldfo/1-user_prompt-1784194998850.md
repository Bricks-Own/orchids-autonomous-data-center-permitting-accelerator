Two related fixes in permitOS/src/components/DocumentFactory.jsx:

PART 1 — Section title text should stay white, not colored, even on a tinted background

The Air/Water/Building/Power Permit Documents CardTitles currently use style={{ color: 'var(--color-chart-N)' }} — the title text itself is colored to match the chart color. Using the lime/emerald chart palette for backgrounds and borders is fine, but it shouldn't extend to every piece of text — titles should stay white/neutral for readability, with only the background wash and border carrying the color accent. Icons can stay colored (that's a reasonable accent), but title text and the "X/Y generated" count should be white.

// before
<Wind size={18} weight="duotone" style={{ color: 'var(--color-chart-1)' }} />
<CardTitle className="text-sm normal-case tracking-normal font-sans" style={{ color: 'var(--color-chart-1)' }}>Air Permit Documents</CardTitle>
...
<span className="text-xs" style={{ color: 'var(--color-chart-1)' }}>{airGenerated}/{AIR_DOCS.length} generated</span>

// after
<Wind size={18} weight="duotone" style={{ color: 'var(--color-chart-1)' }} />
<CardTitle className="text-sm normal-case tracking-normal font-sans text-white">Air Permit Documents</CardTitle>
...
<span className="text-xs text-white">{airGenerated}/{AIR_DOCS.length} generated</span>

Apply the same change (keep icon colored, remove the style color override from CardTitle and the generated-count span, use text-white instead) to Water (chart-2), Building (chart-3), and Power (chart-4) headers. Also add the pt-(--card-spacing) fix from before if not already applied — the header content currently sits flush against the top of the colored box with no top padding while the bottom has proper spacing:

<CardHeader className="items-center pt-(--card-spacing) border-b border-[var(--color-chart-1)]/30 bg-[var(--color-chart-1)]/5">

PART 2 — Apply the same principle to the "All Permits Required" tab

The category headers there (Federal Air Permits & Regulations, Federal Water Permits & Regulations, etc.) use a separate CAT_COLOR map with arbitrary colors unrelated to the theme (indigo, blue, violet, cyan, amber, red) applied to BOTH background and text via the same class string — meaning category title text is colored there too, and using colors that don't even match the lime/emerald palette used everywhere else.

Replace CAT_COLOR entirely. Derive the styling from each category's existing permitKey (already present on every UPSIZED_PERMITS entry from the earlier filtering fix) instead of the separate color field, reusing the same chart-color convention as the Documents tab: air -> chart-1, water -> chart-2, landuse -> chart-3 (matches Building), always (Fire/Safety) -> chart-4 (matches Power). Background and border keep the chart-color tint; category title text goes white, matching Part 1:

const CAT_STYLE = {

  landuse:  { border: 'border-[var(--color-chart-3)]/30', bg: 'bg-[var(--color-chart-3)]/5' },
  always:   { border: 'border-[var(--color-chart-4)]/30', bg: 'bg-[var(--color-chart-4)]/5' },
};

Update the category card and header divs to use CAT_STYLE[cat.permitKey]?.border / .bg for border/background, and text-white for the category title span instead of the old shared arbitrary-color class.
