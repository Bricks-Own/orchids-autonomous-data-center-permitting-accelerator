● Brick PermitOS — Production Readiness

  Verdict: a polished demo, not a product. Don't put it in front of paying customers yet. Audit found 96 verified issues (20 critical, 52 high). Three classes of risk: it doesn't run/deploy, it has zero
  security, and it puts fabricated content into government filings.

  Main issues

  - :red_circle: Backend doesn't boot — node server.js crashes on a missing-module error (frontend source reused under Node ESM with extensionless imports). The API has never actually run as committed.
  - :red_circle: Zero security — no authentication on any endpoint, CORS wide open, no HTTPS, error details leaked to clients.
  - :red_circle: No multi-tenancy + trivial IDOR — one shared DB, sequential IDs, no ownership checks → any visitor can read every customer's site data and chats.
  - :red_circle: The "AI" is fake — no LLM anywhere; it's keyword if/else templates. "RAG vector store" is TF-IDF with an unused embedding column. "Digital Twin / Autonomous Agent" don't exist.
  - :red_circle: Wrong & fabricated domain content — uniform efficiency % applied to all pollutants incl. CO₂; genset emissions ~70% underestimated; AERMOD "results" and BACT/RBLC tables are hardcoded fiction;
  PSD/SER/Title V logic incorrect; live bugs (synthetic-minor always "viable," GHGRP off by 2.5k tpy).
  - :red_circle: "100% EPA Compliant" + "one-click submission" — unsupportable claims that invite customers to file defective documents (your liability).
  - :red_circle: Won't scale / isn't deployed — TF-IDF recomputed every request, stale cache; SQLite incompatible with serverless; vercel.json ships only the frontend, so /api 404s in prod; only 2 of 10 components
  are wired to the backend.
  - :red_circle: No ops foundation — zero tests, no logging, no migrations, no Dockerfile; node_modules and the .db committed to git.
  - :large_green_circle: What's fine — clean code, sensible DB schema, graceful API fallback, draft disclaimers present, solid CFR/state regulatory catalog.
