Part 2 — Complete the "only show what's needed" feature: Building and Power have no document catalog at all
Air and Water documents already correctly filter by what the site's Site Intake Q&A says is needed (showAir/showWater), and the "All Permits Required" tab filters all four types. But DocumentFactory.jsx only has AIR_DOCS and WATER_DOCS — there's no BUILDING_DOCS or POWER_DOCS at all, so a site that needs Building or Power permits but not Air/Water currently shows zero documents, which defeats the whole point.

Good news: real regulatory content already exists for both in permitOS/src/data/permitData.js — BUILDING_MODULES (8 items, IBC/NFPA/local code citations, real deliverables lists) and POWER_MODULES (7 items, FERC/NERC/utility interconnection citations, real deliverables lists) are already fully written out, just not wired into Document Factory as generatable documents.

Add BUILDING_DOCS and POWER_DOCS arrays in DocumentFactory.jsx, following the exact same shape as AIR_DOCS/WATER_DOCS ({ id, key, name, cfr, pages }), sourced from BUILDING_MODULES/POWER_MODULES:

import { BUILDING_MODULES, POWER_MODULES } from '../data/permitData';

const BUILDING_DOCS = BUILDING_MODULES.map((m, i) => ({
  id: i + 1,
  key: `building_${i + 1}`,
  name: m.title,
  cfr: m.regulation,
  pages: '8–15', // reasonable estimate consistent with AIR_DOCS/WATER_DOCS page ranges
}));

const POWER_DOCS = POWER_MODULES.map((m, i) => ({
  id: i + 1,
  key: `power_${i + 1}`,
  name: m.title,
  cfr: m.regulation,
  pages: '8–15',
}));
Add corresponding entries to REG_CITATIONS for each building_N/power_N key (agency from m.agency, type 'Building'/'Power'), so the source/regulation tooltip behavior already built for Air/Water docs works identically for these.
