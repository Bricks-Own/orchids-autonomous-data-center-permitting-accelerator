Two systemic bugs found reviewing the "revamp" commit — both need fixing across every file this pass touched, not just Site Intake.

BUG 1 — Rounded corners are missing almost everywhere. The design spec required Card radius 16px, Input/Select/Button radius 8px, Badge/pill radius 20px, Stepper circles fully round (border-radius 50%). Checked the actual code: src/components/SiteIntake.jsx has exactly ONE "rounded" class in the entire 940-line file (a checkbox), and it's unrelated to any Card/Input/Select/Button/Badge/SectionHeading component. src/components/AuthModal.jsx (the login screen) has ZERO "rounded" classes anywhere. src/components/Header.jsx has only 2. Meanwhile the real shadcn primitives that got correctly scaffolded (src/components/ui/button.jsx, ui/card.jsx) DO have proper rounded classes — so the theme/radius token (--radius: 0.625rem, plus --radius-lg/md/sm derived from it, already defined correctly in src/index.css) exists and works, it's just not being referenced by the hand-rolled components in SiteIntake.jsx/Header.jsx/AuthModal.jsx. The className strings in these files have literal double-spaces exactly where a rounded-* class appears to have been dropped (e.g. Card: `"bg-background border border-border  p-7"` — note the double space before p-7). Also confirmed in Stepper.jsx: the step-circle div is missing rounded-full entirely, so it will render as a square instead of a circle.

Fix: audit every custom component built in this pass (Card, Field, Input, InputMono, Select, PrimaryButton, SecondaryButton, GhostButton, Badge, SectionHeading in SiteIntake.jsx; Stepper.jsx; and whatever equivalents exist in Header.jsx, AuthModal.jsx, and every other file this revamp touched) and add the correct radius class per the design spec: rounded-2xl (16px) for Cards, rounded-lg or rounded-md (8px) for Input/Select/Button, rounded-full for pills/badges/the Stepper circles. Use the theme's rounded-lg/rounded-md/rounded-sm utilities (which map to the --radius CSS variables already set up) rather than hardcoded pixel values, so it stays consistent with the shadcn token system.

BUG 2 — Custom hand-rolled components instead of the real shadcn primitives. The actual shadcn UI components were correctly installed (src/components/ui/button.jsx, card.jsx, input.jsx, select.jsx, badge.jsx, dialog.jsx, etc. all exist and are properly built). But SiteIntake.jsx doesn't import or use any of them — it defines its OWN local Card/Input/Select/PrimaryButton/SecondaryButton/GhostButton/Badge components from scratch instead (which is how Bug 1 happened — reinventing components is exactly how a class of styling got dropped). This defeats the point of installing shadcn's registry components in the first place: consistency, accessibility, and not having to hand-maintain N slightly-different implementations of the same button.

Fix: go through SiteIntake.jsx (and every other file in this revamp) and replace the locally-defined lookalike components with imports from '@/components/ui/button', '@/components/ui/card', '@/components/ui/input', '@/components/ui/select', '@/components/ui/badge', '@/components/ui/dialog', etc. — the real shadcn components, using their variant props (e.g. <Button variant="secondary">) instead of separate PrimaryButton/SecondaryButton/GhostButton components. Delete the local reimplementations once nothing references them.

Also — for my own awareness, not necessarily something to change: the preset brought in Outfit + Raleway fonts (via @fontsource-variable) instead of the Inter + IBM Plex Mono specified in the original design doc. If that's intentional because it's what the preset itself specifies, that's fine and expected — just confirming this wasn't an accidental substitution.

After fixing both bugs, do a final self-check: grep the whole src/components/ directory (grep -rL "rounded" src/components/*.jsx) for any component file with zero "rounded" references — that's a strong signal something is still missing its radius styling. Every Card/Button/Input/Badge/Dialog in the app should have visibly rounded corners matching the design spec, with zero sharp-square-corner elements anywhere, including the login screen.
