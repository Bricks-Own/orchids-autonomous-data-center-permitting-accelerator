Fix severe horizontal overflow on the Air Permit "Remediation Action Matrix" table
Confirmed live
On the Air Permit detail page (Permits → Air Permit → scroll to "Remediation Action Matrix"), the table blows out to ~1656px wide inside a ~949px container. The excess isn't staying contained in the table's own horizontal scrollbar — it leaks out and drags the entire page (including the sidebar) sideways, producing exactly the broken/overflowing look reported: the whole layout shifts, leaving blank space and a badly broken page.

Root cause
In permitOS/src/components/AirPermitAI.jsx, the Remediation Action Matrix table cells:

<TableCell className="text-muted-foreground max-w-[200px]">{b.brickControl}</TableCell>
<TableCell className="text-amber-300 max-w-[200px]">{b.additionalControls || 'N/A — mitigated by Brick controls'}</TableCell>
use max-w-[200px] with no wrapping rule. HTML tables default to table-layout: auto, under which max-width on a <td> does not reliably constrain column width — the browser sizes columns to fit the content's natural (unwrapped) width first. Since b.brickControl and b.additionalControls are full sentences with no whitespace-normal/break-words, the browser lets those columns grow to fit the entire unbroken string, which is what's blowing the table width out to 1656px.

Fix
Add explicit text wrapping to those two cells (and audit every other TableCell across AirPermitAI.jsx, WaterPermitAI.jsx, BuildingPermitAI.jsx, PowerPermitAI.jsx, and DocumentFactory.jsx for the same pattern — any max-w-[...] on a TableCell without a wrap class has the same latent bug):

<TableCell className="text-muted-foreground max-w-[200px] whitespace-normal break-words">{b.brickControl}</TableCell>
<TableCell className="text-amber-300 max-w-[200px] whitespace-normal break-words">{b.additionalControls || 'N/A — mitigated by Brick controls'}</TableCell>
Additionally, as a defensive guard so a similar overflow anywhere else in the app can never again drag the entire page/sidebar sideways: add overflow-x-hidden to the root layout container in App.jsx (the SidebarInset or the <main> element wrapping renderRoutes()), so any future wide-content bug is contained to a local scrollbar on its own element instead of breaking the whole page layout.

Verify after

Open Permits → Air Permit, scroll to "Remediation Action Matrix".
Confirm the table stays within its card's width — long text in "Applied Brick Control" / "Additional Measures Needed" wraps onto multiple lines instead of forcing the column wider.
Confirm the page itself never scrolls horizontally, and the sidebar never shifts, regardless of table content length.
Spot-check the other three permit pages and Document Factory for the same max-w-[...] + no-wrap pattern on any table cell.
