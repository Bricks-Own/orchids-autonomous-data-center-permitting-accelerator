Part 1 — Document Factory shows Air + Water documents regardless of what the site actually needs
In permitOS/src/components/DocumentFactory.jsx, allDocs (around line 644) unconditionally concatenates AIR_DOCS and WATER_DOCS — every site sees all 16 air documents and all 10 water documents, even if that site's Q&A answers say it doesn't need air or water permits. The two document-table sections in the render (Air table around line 858, Water table around line 888) also render unconditionally using AIR_DOCS/WATER_DOCS directly, not filtered.

Fix: this file already imports usePermitData(), so it has access to inputs. Use the same derivation already used everywhere else in the app (Site Intake, Permits, Milestone Timeline):

const showAir = inputs?.hasOnSiteGeneration !== false;
const showWater = inputs?.hasWaterUse !== false;

const allDocs = [
  ...(showAir ? AIR_DOCS.map(d => ({ ...d, docType: 'air' })) : []),
  ...(showWater ? WATER_DOCS.map(d => ({ ...d, docType: 'water' })) : []),
];
Wrap the entire Air Permit Documents section block in {showAir && (...)} and the entire Water Permit Documents section block in {showWater && (...)}, so a site that only needs Air permits shows only the Air documents table — no Water section at all, not even collapsed/empty.

Note: this file only has document catalogs for Air and Water (AIR_DOCS, WATER_DOCS) — there's no Building or Power document set at all yet. If a site needs Building/Power permits but not Air/Water, this page will currently show nothing.
