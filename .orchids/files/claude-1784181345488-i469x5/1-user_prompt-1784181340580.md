Two fixes needed in permitOS/src/components/SiteIntake.jsx:
FIX 1 — Permit-type derivation ignores the Q&A answer
I answered "No" to the water-use question in the Project Scope Q&A, but "Data Center & Water Systems" still showed in Step 2, and "Water Permit" still showed in Step 3 and on the Permits page. Root cause: the derivation formula is (hasWaterUse !== false || coolingMGD > 0 || waterMGD > 0) && 'water' — the equipment-value fallback always wins because coolingMGD/waterMGD default to nonzero regardless of what's answered. Since hasOnSiteGeneration, hasWaterUse, hasNewConstruction, and hasGridInterconnection are always explicit true/false now (never undefined), that equipment-value fallback isn't needed and actively causes this bug. Simplify derivedPermitTypes to trust the Q&A flags alone:
// before
const derivedPermitTypes = [
  (inputs.hasOnSiteGeneration !== false || inputs.turbines > 0 || inputs.gensetCount > 0) && 'air',
  (inputs.hasWaterUse !== false || inputs.coolingMGD > 0 || inputs.waterMGD > 0) && 'water',
  inputs.hasNewConstruction !== false && 'building',
  inputs.hasGridInterconnection !== false && 'power',
].filter(Boolean);
// after
const derivedPermitTypes = [
  inputs.hasOnSiteGeneration !== false && 'air',
  inputs.hasWaterUse !== false && 'water',
  inputs.hasNewConstruction !== false && 'building',
  inputs.hasGridInterconnection !== false && 'power',
].filter(Boolean);
Apply this same simplification everywhere this exact derivation logic is duplicated — App.jsx's permitTypes computation and Permits.jsx's permitTypes computation both currently have the identical broken formula and need the identical fix.
