I made and verified these fixes locally by running the app and testing each interaction directly (not just checking that it builds). Please apply all of them exactly as shown so both copies stay in sync.

1. permitOS/src/components/SiteIntake.jsx — the real root cause of the broken State dropdown.

This was never a Base UI rendering bug. US_STATES in permitOS/src/data/permitData.js is a flat array of strings (['Alabama', 'Alaska', ...]), but the State <Select> was mapping it as if it were an array of {value, label} objects — so every key, value, and the visible text were all undefined, rendering completely empty items. Fix:

// before
{US_STATES.map(s => (
  <SelectItem key={s.value} value={s.value}>{s.label}</SelectItem>
))}

// after
{US_STATES.map(s => (
  <SelectItem key={s} value={s}>{s}</SelectItem>
))}
Verified live: dropdown now shows real state names, and selecting one correctly cascades address/county/lat/lon defaults and updates Air Quality Area Status.

2. permitOS/src/components/ui/select.jsx — SelectItem, keep the <span> wrapper.

Not the actual cause of the State bug (that was #1), but SelectPrimitive.ItemText from @base-ui/react/select still doesn't reliably render plain children as text in this version, so keep this as a plain <span>:

function SelectItem({ className, children, ...props }) {
  return (
    <SelectPrimitive.Item
      data-slot="select-item"
      className={cn(
        "relative flex w-full cursor-default items-center gap-2.5 rounded-none py-2 pe-8 ps-3 text-sm outline-hidden select-none text-popover-foreground focus:bg-accent focus:text-accent-foreground not-data-[variant=destructive]:focus:**:text-accent-foreground data-disabled:pointer-events-none data-disabled:opacity-50 [&_svg]:pointer-events-none [&_svg]:shrink-0 [&_svg:not([class*='size-'])]:size-3.5",
        className
      )}
      {...props}>
      <span className="flex flex-1 shrink-0 gap-2 whitespace-nowrap">
        {children}
      </span>
      <SelectPrimitive.ItemIndicator
        render={
          <span className="pointer-events-none absolute end-2 flex size-4 items-center justify-center" />
        }>
        <CheckIcon className="pointer-events-none" />
      </SelectPrimitive.ItemIndicator>
    </SelectPrimitive.Item>
  );
}
Do not revert this to SelectPrimitive.ItemText in any future pass, including passes that reference the scaffolded shadcn reference project for styling parity.

3. permitOS/src/components/ui/sidebar.jsx — SidebarSeparator was overflowing past the sidebar's right edge.

Base UI's Separator applies data-horizontal:w-full (width: 100% of parent), and combined with mx-2 margin this pushed every separator 8px past the sidebar boundary — visible as the separator line bleeding out past the sidebar into the main content. Fix:

// before
function SidebarSeparator({ className, ...props }) {
  return (
    <Separator
      data-slot="sidebar-separator"
      data-sidebar="separator"
      className={cn("mx-2 w-auto bg-sidebar-border", className)}
      {...props} />
  );
}

// after
function SidebarSeparator({ className, ...props }) {
  return (
    <Separator
      data-slot="sidebar-separator"
      data-sidebar="separator"
      className={cn(
        "mx-2 data-horizontal:w-auto! bg-sidebar-border group-data-[collapsible=icon]:mx-1.5",
        className
      )}
      {...props} />
  );
}
4. permitOS/src/components/ui/sidebar.jsx — SidebarGroup padding didn't match the collapsed icon-button size, so icons/separators looked off-center in the collapsed rail.

The collapsed rail is 3rem (48px) wide. The collapsed icon buttons are size-9 (36px), but SidebarGroup's p-2 (8px each side) only left 32px of content width — 4px too narrow, so icons overflowed asymmetrically. Fix:

// before
className={cn("relative flex w-full min-w-0 flex-col p-2", className)}

// after
className={cn(
  "relative flex w-full min-w-0 flex-col p-2 group-data-[collapsible=icon]:px-1.5",
  className
)}
(This pairs with fix #3's group-data-[collapsible=icon]:mx-1.5 on the separator — both now use 6px insets in collapsed mode, giving a symmetric 36px content width matching the 36px icon buttons exactly.)

5. permitOS/src/components/Header.jsx — top bar height didn't match the sidebar header height, so their border lines didn't align.

Sidebar header renders at exactly 60px; the top bar was 57px (py-3 padding). Fix:

// before
