I've reviewed the full codebase. Here's my understanding of the repository.

  What it is

  Brick PermitOS — a single-page React demo app (a sales/investor-facing prototype) for an "Autonomous Data Center Permitting & Compliance Platform." It pitches an AI tool that replaces
  environmental-permitting consultants for gas-fired hyperscale data centers, covering air + water permits from site intake through issuance and post-permit compliance.

  Stack & structure

  - React 19 + Vite 8 + Tailwind 3, charts via recharts. No backend, no tests, no TypeScript, no router.
  - Actual app lives in permitOS/ (root just has vercel.json + an empty package-lock). vercel.json likely sets the build/output dir.
  - State is entirely local in App.jsx: activeTab, inputs (site/equipment params), and results. A tab switch renders one of 10 feature components. No persistence; refresh resets everything.

  How it works (data flow)

  1. SiteIntake — form for turbines, gensets, water, location. "Run Screening" does a fake 1.2s setTimeout, then calls calcPTE(inputs) and stores the result. Until that runs, the analysis tabs show empty
  states.
  2. utils/calculations.js — the only real "engine":
    - calcPTE — emissions (NOx/CO/SO₂/PM/VOC/CO₂e/HAP) in tpy from heat input × AP-42-style factors, adds genset contribution, applies a flat brickSavings % reduction for "controlled" PTE, then derives
  permit pathway flags (PSD/Title V/synthetic-minor) and a water balance.
    - Plus simulate24h (digital-twin load profile via sine curves), calcRiskScore, buildTimeline, calcTimelineAcceleration.
  3. data/permitData.js — static catalogs: air/water CFR modules, emission factors, 50-state attainment map, milestone phases, other permits.
  4. Feature tabs (AirPermitAI, WaterPermitAI, MilestoneTimeline, DigitalTwin, ComplianceOS, RegulatorCopilot, ExecutiveSummary, Overview) — mostly presentational, reading results + inputs and a lot of
  hardcoded marketing/regulatory copy.
  5. DocumentFactory + utils/documentGenerator.js (1,500 lines) — generates 26 permit documents (16 air, 10 water) as structured text from inputs/results, previewable in a modal and downloadable as a
  single .txt blob client-side.

  Things worth noting

  - It's a demo, not a real permitting tool — emission math is simplified/illustrative, "AI" actions are setTimeout fakes, and "100% EPA Compliant / one-click submission" are marketing claims, not
  implemented integrations.
  - A couple of small inconsistencies (e.g. buildTimeline exists but MilestoneTimeline uses a separate hardcoded Gantt array; defaultInputs is duplicated in both App.jsx and SiteIntake.jsx).
