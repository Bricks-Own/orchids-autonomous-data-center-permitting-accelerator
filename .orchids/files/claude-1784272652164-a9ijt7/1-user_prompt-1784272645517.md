1. Fix: Custom Scenario Builder sliders in Site Planner don't respond to drag
Confirmed live: on Site Planner's "Custom Builder" tab (ScenarioTest component in SitePlanner.jsx), none of the six sliders (Gas Turbines, MW per Turbine, Operating Hours/yr, Brick Savings, Backup Gensets, Genset Hours/yr) respond to drag — tested with precise, correctly-targeted, real trusted pointer events and the underlying value never changes.

The exact same Slider component (permitOS/src/components/ui/slider.jsx, a Base UI slider) works correctly elsewhere — e.g. Site Intake's "Brick Load Reduction" slider (SiteIntake.jsx ~line 397) responds to drag normally. The DOM structure of both (including the hidden native <input type="range"> Base UI renders for accessibility) is identical between the working and broken instances, so this isn't a slider-usage bug — something about the Site Planner context is blocking the interaction.

Investigate these specific hypotheses, in order of likelihood:

CSS leak from the Leaflet map. SitePlanner.jsx imports 'leaflet/dist/leaflet.css' at the top of the file — this is a page-wide import, not scoped to just the map container. Check whether any Leaflet CSS rule (e.g. touch-action, pointer-events, or user-select overrides) is unintentionally matching elements outside the actual map, including the sliders in the adjacent "Custom Builder" tab panel.
Stacking context / pointer-events issue from the Tabs wrapper. Check if the TabsContent panel or an ancestor Card has any CSS (pointer-events: none, a transform creating an unexpected stacking context, or touch-action set at a level that swallows drag gestures) that differs between the Site Planner page and Site Intake page.
Check whether ScenarioTest's own useEffect (the one syncing local params state to inputs prop changes) is somehow re-firing on every pointer move and resetting local slider state mid-drag — even though the code has a JSON.stringify equality guard, confirm this guard is actually preventing re-renders during drag.
Once fixed, verify: drag each of the six sliders, confirm the displayed value updates live, confirm the "Instant PTE Results" panel recalculates, and confirm clicking "Apply Parameters to Site Intake" carries the actual dragged values into Site Intake's Step 2 fields.
